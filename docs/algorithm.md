# Maritime Patrol AI - 알고리즘 설계

이 문서는 현재 시스템의 구조를 유지하면서, **목표 알고리즘 방향**을 정의합니다.  
구현 전 이해를 맞추기 위한 명세입니다.

---

## 1. 전체 구조 (유지)

- **3개 레이어**로 격자맵 구성 → 가중합으로 **복합영향도(순찰 필요성)** 도출
- 격자 기반 공간 + **Q-Learning(또는 경로 생성)** 으로 순찰 경로 산출
- **추론 API**: 요청 시점/구간에 맞는 순찰 필요 지역 + 경로 반환

현재와 동일하게 **레이어 수(3)**, **격자**, **POI(순찰 필요 고도)**, **경로** 개념은 유지합니다.

---

## 2. 레이어의 의미 변경 (현재 → 목표)

### 2.1 현재 (As-Is)

- **3개 레이어**: 정적위험(static), 동적위험(dynamic), 환경제약(weather)
- 각 레이어 값은 **전부 랜덤**으로 생성됨
- 복합영향도 = `static + dynamic + weight * weather` → 이 맵으로 POI 결정 후 순회 경로 학습

### 2.2 목표 (To-Be)

- **동일한 3개 레이어 구조**를 사용하되, 각 레이어는 **실제 데이터**와 **기존 순찰 경로** 간의 **연관성을 AI가 분석해 만든 “관계성 레이어”**로 채움.
- 데이터 소스 예:
  - **환경 요소** (해류, 수심, 조석 등)
  - **날씨**
  - **법령** (통항금지구역, 속도제한 등)
  - **사고 지역 / 사고 이력**
  - **GIS**
  - **AIS** (선박 이동, 밀집도 등)
- 즉, “랜덤 맵 3개”가 아니라 **“데이터 유형별로 순찰 경로와의 관계를 학습한 레이어맵”** 이렇게 바꾸는 것이 목표입니다.

### 2.3 현재 구현 방향 (관계성 레이어)

- **당분간** 관계성 레이어는 **랜덤**으로 구성한다. 나중에 AI 학습 방법으로 “데이터–순찰경로 연관성”을 학습해 레이어를 만들어 넣을 예정이다.
- **중요**: 지금 랜덤으로 만드는 레이어의 **데이터 포맷**은, 이후 AI가 생성할 레이어와 **동일**해야 한다. 포맷을 미리 정해 두고 랜덤 구현으로 채우면, 나중에 AI 출력으로 교체만 하면 된다.

---

## 3. 학습의 정의

학습은 **여러 데이터와 순찰 경로를 비교**하여, **데이터 값 → 순찰 경로 영향**을 나타내는 **그래프(모델)** 와 **그래프별 가중치**를 만드는 과정이다.

### 3.1 데이터별 그래프 학습

- 각 데이터 유형(예: **surface current**)에 대해, **데이터 수치와 순찰 경로의 관계**를 학습한다.
- 그 결과로 **“데이터 값 → 순찰 경로에 미치는 영향”** 그래프를 만든다.
- **예 (surface current, 노트 단위)**  
  - 0~2노트: 순찰 경로에 거의 영향 없음 (그래프 값 낮음).  
  - 2노트부터: 서서히 영향 증가.  
  - 4노트 이상: 해당 위치 주변으로 순찰 경로가 생기는 수준 → 그래프 값 크게 상승.  
  - 6노트 이상: “다른 배가 가지 않아야 하는지” 등 그 근처까지 반드시 순찰해야 하는 수준 → 그래프 값 매우 높게 상승.
- **데이터 유형마다** 위와 같은 **그래프(함수)** 를 학습을 통해 하나의 모델로 만든다.

### 3.2 그래프별 가중치 학습

- 각 그래프에서 나오는 값들은 정보가 많으므로, **그래프별 가중치**를 또 학습한다.
- “그래프 값이 주어졌을 때 그 가중치를 적용하는” 방식으로 **가중치 맵(모델)** 을 학습해 낸다.

### 3.3 학습 결과물

- **데이터별 그래프** + **데이터별 가중치** 에 대한 **학습 맵(모델)** 이 최종 산출물이다.
- 즉, **모든 데이터별 그래프 + 데이터별 가중치** 가 학습을 통해 정의된 하나의 “학습 맵”이 된다.

---

## 4. 추론의 정의

추론은 **학습된 그래프와 가중치**를 사용해, **해당 시간대 데이터**로부터 **순찰 필요성 격자**와 **순찰 지역**을 도출하는 과정이다.

### 4.1 추론 단계

1. **학습된 그래프(모델)** 를 로드한다.
2. **해당 시간대의 데이터**를 가져온다.
3. **그래프 값으로 데이터 값을 치환**한다. (각 데이터 유형별 그래프에 raw 데이터를 넣어 “순찰 영향도” 값으로 변환)
4. 치환된 값을 **격자(grid)** 로 만든다. (데이터별로 격자맵이 나옴)
5. 격자로 만들어진 결과에 **데이터별 가중치 맵**을 적용하여 **최종 순찰 필요성**을 도출한다.  
   이때 **최종 순찰 필요성에 대한 설명**을 계속 **누적**해 낸다.
6. 위 과정을 **순찰 시간대별**로 수행하고, 각 시간대마다 **순찰 지역**을 도출한다.

### 4.2 추론 결과

- **시간대별 순찰 필요성 격자**  
- **시간대별 순찰 지역** (어디를 순찰해야 하는지)  
- **순찰 필요성에 대한 설명** (누적된 설명)  
- 필요 시 **시간 순서에 맞는 순찰 경로** 제공

## 5. 경로 생성 알고리즘

- **우선순위(연료 효율 vs 단시간 등)** 는 당장 중요하지 않다. 무엇을 우선할지는 지금 단계에서는 두지 않는다.
- 순찰 시간이 남으면 **동일 시간대에 여러 구역**을 방문할 수 있다고 본다.
- 경로 생성의 원칙은 다음 두 가지다.
  1. **안전**: 절대적으로 지켜야 할 것.
     - **육지·장애물 등과의 충돌** 회피 (부딪히는 경우 피해가야 함).
     - **선박 밀집 구역 진입 회피** (그물 등에 영향을 받을 수 있음).
  2. **단시간**: 안전한 범위 안에서 **가능한 한 짧은 시간**에 갈 수 있는 경로.
- 연료 효율, “효율 우선” vs “안전 우선” 같은 **경로 산출용 옵션**은 두지 않는다. 무조건 **안전 + 단시간** 기준으로만 경로를 낸다.
- 경로를 만들 때 **제약(육지, 통항금지, 밀집 구역 등)** 을 반영하려면 **GIS 정보**가 필요하다. GIS 데이터는 추후 제공 예정이므로, 당장은 제약 반영 방식을 문서로만 정리해 두고, 포맷/위치만 맞춰 두면 된다.

---

## 6. UI 변경 사항

- **경로 산출 관련 옵션 제거**: 웹 UI에서 “전략”(안전 우선 / 효율 우선 / 광역 감시) 등 **경로 산출 옵션은 제거**한다. 경로는 위 5장 기준(안전 + 단시간)으로만 생성되므로 옵션 불필요.
- **시간 입력 추가**: 순찰 구간·시간대를 다루려면 **시간(또는 시간대)** 입력이 필요하다. 웹 UI에 **시간 입력**을 추가한다.

## 7. 현재 플로우와의 대응

| 구분 | 현재 (As-Is) | 목표 (To-Be) |
|------|----------------|--------------|
| 레이어 입력 | 3개 레이어 모두 랜덤 | 3개 레이어 = 데이터 유형별 **관계성 레이어**(AI 분석 결과). **당분간은 랜덤**, 포맷은 AI와 동일 |
| 레이어 생성 | 시드 기반 난수 | **현재**: 랜덤(포맷 고정). **이후**: 실제 데이터 + 순찰 경로로 **학습된 관계성 모델**로 생성 |
| 학습 목표 | POI 방문 순서 학습 | **데이터별 “값→영향” 그래프** 학습 + **그래프별 가중치** 학습 → **학습 맵(모델)** 구축 |
| 경로 생성 | 전략(안전/효율/광역) 옵션 있음 | **안전 + 단시간** 고정. 옵션 없음. 제약은 GIS 추후 반영 |
| 추론 입력 | 요청 1건 (시간 구간 등) | **1시간 단위 데이터** + 시간 입력 |
| 추론 출력 | 순찰 필요 지역(POI) + 한 경로 | **그래프 로드 → 시간대 데이터→그래프 치환→grid→가중치 적용** → 시간대별 순찰 필요성·순찰 지역·설명 누적·시간순 경로 |
| 웹 UI | 전략 선택 등 경로 옵션 있음 | 경로 산출 옵션 **제거**, **시간 입력** 추가 |

## 8. 정리

- **구조**: 지금처럼 3개 레이어, 격자, POI, 경로 개념 유지.
- **관계성 레이어**: 당분간 **랜덤**으로 구성하되, **데이터 포맷은 이후 AI가 만드는 레이어와 동일**하게 둔다. 나중에 AI 학습으로 교체.
- **경로**: 연료·효율 등 우선순위 옵션 없음. **안전(육지/충돌·선박 밀집 회피) + 단시간** 기준만 사용. GIS 제약은 추후 반영.
- **학습(향후)**: 데이터와 순찰 경로 비교 → 데이터별 **“값→영향” 그래프** 학습 + **그래프별 가중치** 학습 → **학습 맵(모델)** 구축.
- **추론**: 학습된 그래프 로드 → 시간대 데이터 → 그래프로 치환 → grid 구성 → 가중치 적용 → **최종 순찰 필요성** 및 설명 누적 → **시간대별 순찰 지역** 도출 및 시간순 경로 제공.
- **UI**: 경로 산출 옵션(전략 등) 제거, **시간 입력** 추가.

**현재 구현**: 학습(그래프·가중치)은 미구현이며, 추론에서는 **랜덤 생성한 학습 맵**을 `models/patrol_learning_map.npz`에 저장·로드해 사용한다. 시간대별 랜덤 데이터 → 그래프 치환 → 격자 → 가중치 적용 → 순찰 구역·경로까지는 구현되어 있다.
